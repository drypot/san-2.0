   들어가는 글
   -----------

   이 글은 산의 환경 설정 방법을 설명합니다. 프린터로 출력하신
   후 참고하시기 바랍니다.


   산 환경 설정 1
   --------------

   산 환경 설정을 마치면 일단 사용이 가능해 집니다. 메모리 관련 부분
   설정에 대해서는 다음 절을 참고해 주십시요.

   * 설치후 디렉토리 모습

     --+-
       |
       +-SAN--+--FONT
              |
              +--MCPAGE
              |
              +--MACRO
              |
              +--BINDOS


     SAN\         산의 환경 설정 화일과 설명 화일등이 위치합니다.
     SAN\FONT     화면 출력 글꼴이 위치합니다.
     SAN\MCPAGE   문자 입력을 위한 키보드 테이블이 위치합니다.
     SAN\MACRO    매크로 소스와 컴파일된 매크로 바이너리가 위치합니다.
     SAN\BINDOS   도스용 실행화일들이 위치합니다.

   * SAN 환경 변수에 산을 설치한 디렉토리를 지정해야 합니다.

     산 실행화일(SAN.EXE)은 이 환경 변수를 통해서 환경 설정화일(SAN.CFG)을
     참조 합니다.

     예) set SAN=C:\UTIL\SAN\

   * PATH 환경 변수에 BINDOS 디렉토리의 패스를 추가합니다.

     BINDOS 디렉토리를 PATH 변수에 추가해 놓으면, 현재 위치해있는
     디렉토리에 상관없이 산을 기동할 수 있습니다.

     단, 매크로 컴파일러는 MC.EXE 를 실행시키지 마시고, SAN 디렉토리에서
     MC.BAT 화일을 실행시키시기 바랍니다.

   * SAN.CFG 환경 설정 화일을 수정해야 합니다.

     산 꾸러미에 들어있는 SAN.CFG 화일은 이미 적당한 초기값을 가지고 있습니다.
     이중에서 사용하시는 자판에 해당하는 부분은 자신의 환경에 맞도록 재 설정
     하셔야 합니다.

     산은 기동시 키보드 입력용 테이블 (meta code page) 화일을 읽어서
     사용 자판을 선택합니다.

     초기치는 두벌식과, 쿼티자판입니다. (일반적으로 사용되는
     자판에 그려져 있는 한글, 영문 자판 배열이 두벌식과 쿼티입니다.)

     세벌식 한글 자판 사용자나, 드보락 영문자판 사용자는 다음과 같이
     두벌식, 쿼티 지정행을 주석화 해서 생략시키시고, 세벌식, 드보락
     지정행의 주석 표시를 없애시기 바랍니다.

     예)

     초기 형태

     meta_code_page          -\mcpage\qwerty.mcp
     #meta_code_page         -\mcpage\dvorak.mcp
     meta_code_page          -\mcpage\han3-90.mcp
     #meta_code_page         -\mcpage\han2.mcp

     세벌식, 드보락 자판으로 수정된 형태

     #meta_code_page          -\mcpage\qwerty.mcp
     meta_code_page         -\mcpage\dvorak.mcp
     #meta_code_page          -\mcpage\han3-90.mcp
     meta_code_page         -\mcpage\han2.mcp

   * 기동

     명령행에서 SAN 이라 입력하시면 몇초후 편집화면이 나타나면서
     에디터가 실행됩니다.

     수행에 문제가 있다면, 위 과정을 다시 점검해 주시고,
     README.TXT 화일에 적혀있는 제작자 연락처로 연락 주시기 바랍니다.

   * 설정 예

     만약 산 꾸러미를 C:\SAN\ 디렉토리에 풀어 놓으셨다면,
     다음 두 행을 AUTOEXEC.BAT 화일에 추가하시고 컴퓨터를 다시
     부팅시켜 주십시요. 부팅후, 명령행 상에서 SAN 이라고
     입력하시면 됩니다.

     set SAN=C:\SAN\
     set PATH=%PATH%;C:\SAN\BINDOS


   도스 익스텐더 환경 설정
   -----------------------

   산 2.2 판은 자체 제작해서 사용하던 기존 가상 메모리 관리기 대신에
   DOS/4GW 도스 익스텐더를 사용합니다. 이 소프트웨어와 사용되도록
   설계된 프로그램은 도스환경에서 수행시 작착되어 있는 모든 램을
   효율적으로 사용할 수 있고, 사용자가 원할 경우 32M 까지의 가상
   메모리를 사용할 수 있습니다.

   쪽지 설명: 가상메모리
   기존의 도스 환경에서는 물리적으로 설치되어 있는 램이 부족할 경우 큰
   데이터를 처리하지 못했습니다. 산 1.9 부터 사용된 가상 메모리
   기법이란, 이 문제를 해결하기 위해서, 하드 디스크의 빈 공간을 임시
   저장 장소로 이용해서 마치 많은 물리적 램을 설치한 효과를 내는
   것입니다. 사용자의 시스템에 8메가 바이트의 램이 설치되어 있고, 하드
   디스크 여유가 있다면, 가상 메모리 크기를 16 메가로 설정해서 마치
   16메가 바이트의 램이 설치된 시스템처럼 사용할 수 있게 됩니다.

   가상 메모리를 사용하기 위해서는 vmm.bat 화일을 실행한 후 산을
   기동해야 합니다. vmm.bat 화일은 DOS4GVM 환경 변수를 설정하는데 이
   환경 변수는 DOS/4GW 도스 익스텐더가 사용하는 환경 설정 화일의
   패스를 가집니다.

   참고: 가상 메모리 기법을 사용하면, 산의 기동시 초기화 시간이 더
   길게 걸립니다. 그리고, 가상 메모리 관리 능력이 32MByte 라고는
   하지만, 4 메가나 8 메가 시스템에서 수십메가짜리 화일을 처리하는
   것은 하드 디스크 입출력에 시간을 많이 허비하기 때문에 효율적이지
   못합니다.


   DOS/4GW 가상 메모리 설정 화일
   -----------------------------

   SAN.VMC 화일에는 다음과 같은 항목이 있고, 적절한 초기치가 지정되어
   있습니다.

   주석은 느낌표(!)로 시작합니다. 예로 DELETESWAP 기능을 막기 위해서는
   !DELETESWAP 이라 표시하시면 됩니다.

   그리고 중간에 공백행을 넣지 마시기 바랍니다. 처음 공백행에서 환경
   설정 화일 읽기를 마칩니다. 수치 단위는 모두 킬로바이트 입니다.


   MINMEM = 2048             프로그램 수행을 위한 최소 램 요구량입니다.

   MAXMEM = 4096             최대 램 사용량입니다. 산에서 확장 메모리를
                             사용하는 외부 프로그램을 실행시키기 위해서
                             메모리 사용을 제한할 수 있습니다.

   SWAPMIN = 2048            기동시 확보하는 하드디스크 메모리량입니다.

   SWAPINC = 2048            확보한 디스크 메모리를 모두 사용했을 경우
                             단계적으로 추가 확보하는 메모리량 입니다.

   SWAPNAME = \DOS4GVM.SWP   디스크 메모리용 화일의 패스입니다.

   DELETESWAP                프로그램 수행이 끝나면, 디스크 메모리용 화일을
                             지웁니다.

   VIRTUALSIZE = 16384       최대 가상 메모리 크기 입니다.


   사용시 제한
   -----------

   산은 모든 확장 메모리를 사용하며, 가상 메모리 사용시 디스크 영역을
   보조 메모리로 사용할 수 있습니다. 메모리 사용량은 최대 32
   메가입니다.

   사용중 메모리가 부족하면 프로그램이 비정상 종료합니다. 특히 UNDO
   기능 사용시는 입력이 없더라도, 커서 위치 기록등으로 메모리를 계속
   소모하므로 주의하시기 바랍니다. (UNDO 기능 참고)

   한 화일이 가질 수 있는 줄 수, 한 줄이 가질 수 있는 문자수, 동시에
   편집 가능한 화일 수 등의 제한은 2Giga 입니다. 그 이상의 자료를
   처리하려는 시도에 대한 작동은 정의되어있지 않습니다.

   UNDO 버퍼 크기는 환경 설정화일을 통해서 지정합니다. 초기치는 64KB
   입니다. 이 크기는 각 화일별 버퍼 크기 입니다. 메모리가 충분하고,
   많은 화일을 한꺼번에 처리하지 않는 경우라면 1MB 이상 설정하는 것도
   가능합니다.


   UNDO
   ----

   이 기능은 2.2 판에서 새로 지원되는 것입니다. 새 화일 작업을
   시작하는 경우나, 화일을 읽어들여 작업을 시작하는 순간 부터 사용자가
   문서에 가하는 모든 수정이나, 커서이동은 undo 버퍼에 기록됩니다. 이
   자료를 통해서 편집중 실수로 지운 내용을 복구할 경우와 같이 문서에
   가한 수정을 되돌릴 수 있습니다.

   편집 내용을 되돌리던 도중 다시 원상 복귀하는 작업, 즉, undo 의 undo
   는 redo 라 하며, undo 도중에 사용자가 커서를 움직이거나 문서 내용을
   수정하지 않았다면 redo 가 가능합니다.

   undo/redo 기능은 마치 카셋 테이브를 앞으로 감아 듣고, 뒤로 풀어
   듣는 동작과 비슷합니다.

   아직 기능이 완전하지 못해서 커서 이동과 문서내용 변경만 undo 할 수
   있습니다. 문서 중 표식(mark)나 블럭 설정은 다시 살려 내지 못합니다.
   즉, 블럭을 설정해서 화일의 일 부분을 삭제 하셨다면, undo 후에,
   삭제된 내용은 복구되나 블럭 설정은 복구되지 않습니다.

   초기 키 설정은 다음과 같습니다.
   undo 기능: Alt-Backspace,
   redo 기능: Alt-Shift-Backspace


   도스 명령행 부르기 / 외부 화일 실행
   -----------------------------------

   산은 기동후 기본 메모리를 50KB 정도 차지하므로, 다른 프로그램
   수행에 충분한 메모리를 제공합니다.

   산 작업 도중 도스 명령행으로 잠시 나가기 위해서는 F10 을 누릅니다.
   프롬프트 앞에 [SAN] 이란 표시가 되며, 도스 프롬프트가 제공될
   것입니다. 다시 산으로 돌아오기 위해서는 exit 명령을 내리면 됩니다.

   산 작업 도중 도스 명령행으로 나가지 않고 외부 프로그램을 실행시키기
   위해서는 F11 을 누릅니다. 화면 아래 입력행에 수행할 명령 이름을
   입력하십시요. 프로그램 수행이 끝나고, 아무 키나 누르면 산 입력
   화면으로 돌아옵니다.

   F5 부터 F8 까지는 왜부 배치화일을 수행하는데 기능이 배당되어
   있습니다. 각각, f5.bat, f6.bat, f7.bat, f8.bat 를 실행합니다.
   f5.bat 와 f6.bat 를 실행한 후에는 'press a key' 라는 메시지나
   나타나지 않습니다. 이 배치화일을 호출할 때는 현재 편집중인 화일명에
   대한 정보를 인자로 전달 합니다. 인자 전달 순서는 각 배치화일을
   참고하시기 바랍니다.
